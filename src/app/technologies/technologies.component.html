Development

I chose angular as my framework to work with due to its growing popularity and ease of use. Since the application has a dashboard feel to it, I wanted to be able to work with different parts of the page, or components, and update them individually. We never learned Angular throughout our degree program, so I also wanted to challenge myself to learn a totally new technology.

Since I chose Angular, I knew that I would be working with Typescript and JavaScript which Node.js can be used to easily install all build and development tools that I would need. I have never worked with or learned Typescript before, so I wanted to challenge myself to learn and familiarize myself with Typescript and NPM.

Much like the other technologies, I chose Visual Studio Code as my IDE due to the fact that I have never used it before. I have also heard from others who have worked with Angular that Visual Studio Code is a great IDE to develop Angular applications.

Throughout our degree program, we have mainly worked with MySQL as our database for projects. For this project, I wanted to utilize a different database and I chose Firebase to gain more experience with the Google Web Services.



API's

At Ticketmaster, most of the status reporting is done via Google Sheets, so I wanted to integrate the Google Sheets API so that users could access information from Google straight through the application.

Box is utilized at Ticketmaster to not only make it easier for employees to save important files, but to share and distribute these files amongst different teams. Being able to integrate Box into the application was a very important feature and I was able to gain great experience working with their API.

Another platform that is widely used to manage work at Ticketmaster is Jira. Almost every team utilizes Jira Software to create backlogs, manage issues, and keep track of their sprints, so because of this I wanted to integrate the Jira API so that users can access their issues straight in the application.



Hosting

I chose Heroku as my hosting solution due to its ease of use, auto-deployment with GitHub, and free-tier usage. After gaining experience deploying to AWS, Azure, GCP, and Heroku, I found Heroku to be the best solution for this project.
