Ticketmaster PMO Application


Introduction

At Ticketmaster, there are many different development teams across 4 separate segments; Enterprise, Marketplace, Resale, and TechOps. Each segment consists of various teams working on a number of different projects. Along with the projects in these different segments, there are also 4 major programs outside of the segments that contain a number of development teams as well. These programs include Secure Ticketing, CCPA, Host Modernization, and Tokenization.


The Problem

With all of these projects and programs and various development teams, status reporting can become very inconsistent and hard to track across the organization. As an intern in Program Management, my summer project was to find a way to consolidate the status reports that are being sent out, and to come up with a new method to maintain consistency across all teams.


The Goal

The goal of this project is to develop a tool that the entire PMO can use to easily track, manage, and share status reports, newsletters, and any other information that is shared across the company. Currently, Ticketmaster uses tools such as Google Docs, Google Sheets, Smartsheet, Confluence, MailChimp, Box, Microsoft Excel, Microsoft Word, and Jira to track, document, create, and distribute status reports for each segment. This project aims to simplify and consolidate these tools into one application, eliminating the need to use different tools and services to accomplish a task.
